## 큰 그림 (장기 비전)

```
OU는 인터넷 위에 올라간 모든 서비스의
공통 레이어(DataNode + DataView)를 선점하는 것이다.

카톡, 인스타, 유튜브, 쿠팡이 각각 따로 만든 것들을
OU는 하나의 아키텍처로 다 담는다.

채팅을 먹으면:
  카카오가 점유한 사업 분야 전부 흡수 가능
  쇼핑, 방송, 페이, 택시, 웹툰, 선물하기...
  → 모두 DataNode + DataView로 일관되게 표현 가능

SNS를 먹으면:
  SNS 광고 시장, 인플루언서 마케팅, 라이브 커머스
  브랜드 계정, 숏폼, 크리에이터 이코노미...
  → 이미 설계된 구독/피드/크리에이터 구조로 흡수

카톡/SNS가 이 사업들을 붙일 때: 기능을 억지로 붙임 (이질감)
OU가 붙일 때: 모두 DataNode + DataView로 일관 (이질감 없음)
```

## 아키텍처 확장성 원칙 (불변)

```
지금 설계하는 것이 나중 확장을 막으면 안 된다.

1. DataNode domain은 확장 가능:
   현재: schedule/task/habit/knowledge/idea/relation/emotion/finance
   추후: product/broadcast/payment/location/... 추가만 하면 됨

2. 뷰 레지스트리는 대등한 뷰 구조 (플러그인이 아님):
   쇼핑뷰/방송뷰/결제뷰/코드뷰/터미널뷰 → 레지스트리 등록만으로 동작
   기존 코드 수정 불필요
   모든 뷰는 동등 — 에디터도, AI도, 터미널도 하나의 뷰
   파일도 뷰다: 파일 시스템 = 가장 원시적인 렌더러

3. 채팅/피드는 DataView 기반:
   어떤 DataNode든 뷰로 만들어 전송/표시 가능
   새 도메인 추가해도 채팅/피드는 자동 지원

단계는 있다:
  지금은 LLM 채팅 + 기본 뷰에 집중
  아키텍처만 확장 가능하게 설계하면
  나머지는 자연스럽게 따라온다
```

## 핵심 개념 (전역 불변)

### 용어

| 용어 | 정의 | 금지 표현 |
|------|------|-----------|
| **DataNode** | 의미 단위로 구조화된 데이터. 그래프의 노드 하나 | 항목, 카드 |
| **데이터뷰 (DataView)** | DataNode를 원하는 형식으로 렌더링한 뷰. = 프론트엔드. | 사전, 목록, 페이지 |
| **그래프뷰** | 노드=별, 엣지=중력으로 시각화한 데이터뷰 | — |
| **회원** | 로그인한 사용자. 개인 DB 보유 | 유저 |
| **관리자** | OU 팀. 생태계 첫 번째 생산자. 회원 중 1번 | — |
| **트리플** | subject-predicate-object. 온톨로지 기본 단위 | — |
| **.ou** | OU 고유 파일 포맷 (DataNode + 뷰 설정) | — |

### 불변 원칙

```
1. 세션 없음. 대화는 연속 스트림으로 쌓인다.
2. 데이터는 삭제되지 않는다. 이벤트로 기록된다.
3. 필터는 존재하는 데이터 기반으로만 생성된다.
4. 원본 파일은 항상 R2에 보존된다. (파싱 실패해도)
5. 관리자 DataNode는 복사 저장 금지. 참조(reference)로만.
6. triples predicate는 표준 11개만 사용. 임의 추가 금지.
7. API 키는 서버사이드 only. 클라이언트 노출 금지.
```

### 데이터 계층

```
messages → data_nodes → sections → sentences
                ↓
            triples ←── triple_sentence_sources ──→ sentences
            (node에 속함)   (출처 연결, 선택적, N:M)
```

한 문장에서 트리플 여러 개 가능 (1:N).
여러 문장을 조합해서 트리플 하나 추론 가능 (N:M).
triples는 sentences의 자식이 아니라 data_nodes의 자식.

### 도메인 → 뷰 힌트

```
schedule → 캘린더뷰  |  task → 칸반뷰    |  habit → 히트맵뷰
knowledge → 그래프뷰 |  idea → 마인드맵뷰 |  relation → 관계그래프뷰
emotion → 저널뷰     |  finance → 차트뷰
```
