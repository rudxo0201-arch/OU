# 접근성 기준

> KRDS 참조: WCAG 2.1 AA 수준 달성

## 명암비 **[DO]**

| 조건 | 최소 명암비 |
|------|-----------|
| 본문 텍스트 (< 18px Regular, < 14px Bold) | **4.5:1** |
| 큰 텍스트 (≥ 18px Regular, ≥ 14px Bold) | **3:1** |
| UI 구성요소 (보더, 아이콘, 포커스 링) | **3:1** |

## 터치 타겟 **[DO]**

| 입력 방식 | 최소 크기 |
|----------|----------|
| 마우스 | 17px × 17px |
| 터치 | **44px × 44px** |

## 키보드 네비게이션 **[DO]**

```
Tab: 다음 포커스 요소로 이동
Shift+Tab: 이전 포커스 요소로 이동
Enter/Space: 버튼/링크 활성화
Escape: 모달/팝업 닫기
Arrow Keys: 메뉴/리스트 내 이동
```

## 포커스 표시 **[DO]**

```
모든 인터랙티브 요소에 포커스 링 표시
포커스 링: 2px solid, 외곽에 위치
색상: 시스템 기본 또는 brand.6
outline-offset: 2px
```

## 건너뛰기 링크 (Skip Link) **[BETTER]**

```
페이지 최상단에 "본문 바로가기" 링크
기본: 화면 밖에 숨김
포커스 시: 화면에 표시
```

## 스크린 리더 **[BETTER]**

```
- 모든 이미지에 alt 텍스트
- 아이콘 버튼에 aria-label
- 동적 콘텐츠 변경 시 aria-live
- 랜드마크 역할 (nav, main, aside, footer)
- 제목 위계 순서 준수 (h1 > h2 > h3)
```

## 색상 의존 금지 **[DO]**

```
정보를 색상만으로 구분하지 않는다.
색상 + 아이콘, 색상 + 텍스트, 색상 + 패턴 병행.
예: 에러 = 빨간색 + ⚠️ 아이콘 + "오류" 텍스트
```
