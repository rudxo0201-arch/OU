# OU UX Flows

> OU 웹앱의 전체 사용자 흐름 문서. 각 플로우는 FigJam 다이어그램과 대응된다.

## 전체 사이트맵

```
/                    Landing (비로그인 홈)
/login               로그인/회원가입
/terms               이용약관/개인정보처리방침
/explore             Explore (로그인: MyUniverse / 비로그인: PublicExplore)
/explore/browse      데이터셋 브라우징
/dictionary          한자 사전
/boncho              본초 사전
/acupoint            경혈 사전
/prescription        방제 사전
/flashcard           플래시카드
/quiz                퀴즈
/stats               학습 통계
/universe            나의 우주
/settings            설정
/admin               관리자
```

---

## Flow 1: 신규 사용자 온보딩

### 경로 A: 직접 탐색
```
Landing(/) → "TRY IT" → /explore(비로그인) → /explore/browse → 사전 둘러보기
```

### 경로 B: 바로 가입
```
Landing(/) → "회원가입" → /login(signup) → 약관 동의 → 가입 완료 → /explore(로그인)
```

### 경로 C: 탐색 후 가입
```
Landing(/) → TRY IT → 사전 탐색 → 관심 생김 → 회원가입 → /explore(로그인)
```

### 핵심 결정
- 비로그인도 사전 탐색 가능 (진입 장벽 낮춤)
- 랜딩에서 3D 그래프 시각화로 시선 끌기
- 로그인 유저는 /explore로 자동 리다이렉트

### 에지 케이스
- 이미 로그인 상태로 / 방문 → /explore로 리다이렉트
- 약관 미동의 시 → 가입 버튼 비활성화

---

## Flow 2: 인증

### 로그인
```
/login → 이메일+비밀번호 입력 → 성공 → /explore
                              → 실패 → 에러 메시지 → 재시도
```

### 회원가입
```
/login(signup 모드) → 이메일+비밀번호 → 약관 동의 체크 → 가입 → /explore
                                    → 약관 보기 → /terms → 돌아와서 체크
```

### Google OAuth
```
/login → Google 버튼 → OAuth 팝업 → 성공 → /explore
                                  → 실패 → 에러 → 재시도
```

### 에지 케이스
- 이메일 중복 → "이미 가입된 이메일" 에러
- 비밀번호 불일치 → "비밀번호가 틀렸습니다" 에러
- OAuth 취소 → 로그인 페이지로 복귀

## Flow 3: 사전 탐색 (4개 사전 공통)

### 공통 패턴
```
사이드바 → 사전 선택 → 그리드뷰 → 검색/필터/정렬 → 카드 클릭 → 사이드패널
```

### 각 사전별 특성

| 사전 | 항목 수 | 고유 필터 | 정렬 옵션 |
|------|--------|----------|----------|
| 한자 /dictionary | 5,175+ | 중요도, 초성 | 이름, 중요도, 획수 |
| 본초 /boncho | 400+ | 카테고리 | 카테고리, 이름, 중요도 |
| 경혈 /acupoint | 360+ | 경락, 체표구역 | 코드, 이름, 경락, 구역 |
| 방제 /prescription | 200+ | 대분류/소분류 | 카테고리, 이름, 출전, 약재 수 |

### 상세 패널 흐름
```
카드 클릭 → 사이드패널(320px) 열림
  → 상세 정보 표시
  → 관련 항목 클릭 → 크로스 링크 (다른 사전으로 이동 가능)
  → 모바일: 바텀시트(85vh)로 전환
```

### 크로스 링크
- 본초 → 한자 (약재명의 한자)
- 방제 → 본초 (구성 약재)
- 경혈 → 한자 (혈명의 한자)

### 에지 케이스
- 검색 결과 없음 → 빈 상태 표시
- 데이터 로딩 → 스켈레톤/로딩 상태
- API 실패 → JSON fallback (한자만)

## Flow 4: 학습 (플래시카드 + 퀴즈)

### 플래시카드 /flashcard
```
STEP 1: 설정
  → 중요도 필터 (★★★/★★/★)
  → 카드 수 (10/20/50)
  → 시작 버튼

STEP 2: 학습
  → 카드 표시 (한자)
  → 뒤집기 (뜻/음 보기)
  → 앎 / 모름 버튼
  → 다음 카드

STEP 3: 결과
  → 정확도 표시
  → 다시 하기 / 통계 보기
```

### 퀴즈 /quiz
```
STEP 1: 설정
  → 모드: 뜻 맞추기 / 음 맞추기
  → 중요도 + 문제 수
  → 시작 버튼

STEP 2: 풀기
  → 문제 표시
  → 4지선다 중 선택
  → 정답/오답 피드백
  → 다음 문제

STEP 3: 결과
  → 정확도 % 표시
  → 다시 하기 / 통계 보기
```

### 통계 /stats
```
4개 메트릭 카드:
  → 학습 한자 수 (전체 5,175 대비)
  → 연속 학습일 (streak)
  → 플래시카드 정확도
  → 퀴즈 정확도
  → 전체 진행도 링
```

### 에지 케이스
- 학습 데이터 없음 → 빈 상태 "아직 학습 기록이 없습니다"
- 중간에 나가기 → 진행 상실 (세션 기반)

## Flow 5: 나의 우주 (Universe)

### 비로그인
```
/universe → Empty State → "회원가입" 버튼 / "둘러보기" 버튼
```

### 로그인
```
/universe → 컬렉션 그리드 (타입 배지 표시)
  → 항목 삭제 → 삭제 확인 → 그리드 갱신
  → 항목 클릭 → 상세 보기
```

### Explore 연동
```
/explore (로그인) → MyUniverseView (개인 컬렉션)
/explore (비로그인) → PublicExploreView → /explore/browse (공식 데이터셋)
/explore/browse → 데이터셋 카드 (한자/본초/경혈/방제)
  → 공개 우주 (Coming Soon)
```

### 에지 케이스
- 컬렉션 비어있음 → 빈 상태 + 사전 탐색 유도
- 인증 만료 → 로그인 리다이렉트

---

## Flow 6: 설정 & 계정

```
/settings (인증 필수)
  → 프로필: 표시 이름 변경, 이메일 확인
  → 외관: 다크/라이트 모드 토글
  → 로그아웃 → 확인 → Landing(/) 이동
```

### 에지 케이스
- 비로그인으로 /settings 접근 → 로그인 리다이렉트
- 이름 변경 실패 → 에러 메시지

---

## Flow 7: 관리자

```
/admin (admin 권한 필수)
  → Tab 1: Dashboard
    → 유저/한자/관리자 수 카드
    → 유저 테이블 → 역할 변경 (user ↔ admin)
  → Tab 2: Hanja DB
    → 검색 + 고급 필터 (초성/중요도/분야/학년/형태/획수)
    → 항목 클릭 → Edit Modal → 저장
    → 항목 삭제 → 확인 Dialog
    → 페이지네이션 (50개/페이지)
```

### 에지 케이스
- 비관리자 접근 → 접근 거부 + 리다이렉트
- 자기 자신 역할 변경 → 방지 또는 경고

## 네비게이션 구조

### 데스크톱 (Sidebar)
```

────────────────

  플래시카드 | 퀴즈 | 통계
```

- 팝업 → 중앙 정렬 모달
