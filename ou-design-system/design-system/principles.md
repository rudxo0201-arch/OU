# 1단계: 디자인 원칙

> OU의 모든 UI/UX 설계 의사결정의 최상위 기준.

## 핵심 가치

> "지식을 소유하다" — 데이터를 나만의 우주로 만드는 경험

## 7가지 원칙

### 1. 심미성 (Aesthetics First)
- **정의**: 시각적 아름다움은 타협 대상이 아니다
- **이유**: OU는 데이터를 "게임"으로 만드는 서비스. 심미성이 핵심 차별점
- **실천**: 성능 문제는 렌더링을 똑똑하게 해서 해결. 품질을 낮추지 않는다

### 2. 재미 (Playfulness)
- **정의**: 학습이 게임처럼 즐거워야 한다
- **이유**: 한의대 데이터 학습은 본질적으로 반복적. 재미가 없으면 이탈
- **실천**: 커스터마이징, 스킨, 탐험, 수집 등 게임 메카닉 적용

### 3. 간결 (Simplicity)
- **정의**: 복잡한 데이터를 단순하게 보여준다
- **이유**: 5,000개 이상의 한자, 400개 본초 — 정보 과부하 방지
- **실천**: 점진적 공개, 단계적 필터링, 깨끗한 레이아웃

### 4. 일관성 (Consistency)
- **정의**: 같은 기능은 어디서나 같은 방식으로 동작한다
- **이유**: 4개 사전, 학습 도구, 우주 뷰 — 패턴 학습 비용 최소화
- **실천**: 공통 컴포넌트, 통일된 인터랙션 패턴

### 5. 소유감 (Ownership)
- **정의**: 사용자가 "나만의 것"을 만들 수 있다
- **이유**: 데이터를 소유하고 꾸미는 경험이 핵심 가치
- **실천**: 뷰 커스텀, 우주 꾸미기, 템플릿 편집

### 6. 포용 (Inclusivity)
- **정의**: 누구나 접근할 수 있다
- **이유**: 접근성은 선택이 아닌 필수
- **실천**: WCAG 2.1 AA 준수, 키보드 네비게이션, 다크모드

### 7. 신뢰 (Trust)
- **정의**: 데이터와 시스템을 신뢰할 수 있다
- **이유**: 학습 데이터의 정확성이 서비스의 생명
- **실천**: 명확한 피드백, 안정적 동작, 데이터 보존
