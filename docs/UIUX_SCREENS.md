# UIUX_SCREENS.md — 전체 화면 상세 명세

> OU의 모든 화면을 사용자 흐름 순서로 정의한다.
> Claude Code가 각 화면 구현 시 이 파일을 참조한다.
> 디자인 토큰은 FRONTEND_DESIGN.md 참조.

---

## 전체 라우트 맵

```
/                   랜딩페이지        비로그인 진입점
/login              로그인/가입
/terms-agree        약관 동의

/chat               OU-Chat          핵심 (비로그인 5턴 허용)
/my                 내 우주           그래프뷰 + 저장된 뷰
/accuracy           정확도 높이기
/settings           설정
/settings/upgrade   구독 업그레이드
/admin              관리자 패널

/view/[nodeId]      원본 파일 뷰어   (Phase 2)
/feed               SNS 채널          (Phase 3)
/messages           OU 채팅           (Phase 3)
```

## 화면 1: 랜딩페이지 `/`

### 목적
비로그인 사용자가 OU를 처음 접하는 화면. 킬러 데모를 직접 경험시킨다.

### 레이아웃
```
┌─────────────────────────────────────────────────┐
│  좌 50%: PixiJS 그래프뷰 (데모 노드들)            │
│  우 50%: 카피 + CTA                              │
│                                                  │
│  그라데이언트: 경계선 → var(--mantine-color-body) │
└─────────────────────────────────────────────────┘
모바일: 그래프 50vh (위) + 카피 50vh (아래) 수직 스택
```

### 콘텐츠
```
Display: "Just talk."
Body:    "일정도, 할 일도, 아이디어도.
          OU가 듣고, 기억하고, 보여줍니다."

CTA 버튼 1 (Primary): "무료로 시작하기"  → Google OAuth
CTA 버튼 2 (Ghost):   "먼저 체험해보기"  → /chat (비로그인)
```

### 그래프뷰 데모 데이터
```
미리 만들어진 데모 DataNode들 표시 (실제 사용자 데이터 아님)
  - "결혼식 일정" (schedule)
  - "독서 기록" (knowledge)
  - "이번 달 지출" (finance)
  - "아이디어 메모" (idea)
노드 클릭 불가. 패닝/줌 가능. 60fps.
```

### 인터랙션
```
[무료로 시작하기] 클릭 → Google OAuth → /my
[먼저 체험해보기] 클릭 → /chat (비로그인, 5턴 제한)
로그인 상태로 접근 → /my 자동 리다이렉트
```

## 화면 2: 로그인 `/login`

### 목적
가입 장벽 최소화. Google OAuth 최우선.

### 레이아웃
```
전체 화면 중앙 카드 (width: 380px)

┌────────────────────────┐
│  OU  (Orbitron 폰트)   │
│  Just talk.            │
│                        │
│  [G Google로 로그인]   │
│                        │
│  ──── 또는 이메일로 ───  │
│                        │
│  이메일 ______________  │
│  비밀번호 ____________  │
│  [로그인 / 가입하기]   │
│                        │
│  계정이 없으신가요? 가입 │
└────────────────────────┘
```

### 상태
```
로그인 모드 (기본)
가입 모드 (토글)
로딩 (API 응답 대기)
에러 (이메일 중복, 비밀번호 틀림 등)
이메일 발송 완료 (가입 후 verify 안내)
```

### 인터랙션
```
Google 버튼 → OAuth → /auth/callback → /my
이메일 가입 → verify 메일 발송 → 안내 메시지
이미 로그인 상태 → /my 리다이렉트
```

## 화면 3: OU-Chat `/chat`

### 목적
OU의 핵심. 말하면 데이터가 된다.

### 레이아웃 (데스크톱)
```
┌── Sidebar ──┬────────────────────────────────┐
│             │  TokenGauge                     │
│             │  ─────────────────────────────  │
│             │  MessageStream (스크롤)          │
│             │                                 │
│             │    [OU 온보딩 메시지]            │
│             │    [사용자 메시지 bubble]        │
│             │    [OU 응답 bubble]              │
│             │      NodeCreatedBadge           │
│             │    ...                          │
│             │                                 │
│             │  ─────────────────────────────  │
│             │  SaveNudge (비로그인 5턴 후)     │
│             │  ─────────────────────────────  │
│             │  ChatInput + 첨부 버튼           │
└─────────────┴────────────────────────────────┘
```

### TokenGauge 상세
```
비로그인:   "N턴 남았어요" + 얇은 프로그레스바
Free:       "N/100 토큰" + 프로그레스바
90% 이상:  yellow-5 경고색
초과:       업그레이드 모달 팝업 (입력 내용 보존)
```

### MessageStream 상세
```
사용자 bubble:
  위치: 우측 정렬
  스타일: Paper (0.5px 보더), padding 12px
  최대 너비: 70%

OU bubble:
  위치: 좌측 정렬
  스타일: 배경 없음 (텍스트만)
  스트리밍 중: 커서 애니메이션 (|)
  스트리밍 완료 0.5초 후: NodeCreatedBadge 등장

NodeCreatedBadge:
  위치: OU bubble 하단 우측
  형태: 작은 pill badge (보더만)
  텍스트: "💫 일정 노드가 생성됐어요"
  등장: 아래서 fadeIn 150ms
  소멸: 10초 후 fadeOut
  클릭: /my 이동

ViewRecommendBadge:
  위치: OU bubble 아래 별도 카드
  등장 조건: domain DataNode 임계값 도달
  형태: Paper 카드 + [네, 보여줘] [괜찮아] 버튼
  [네, 보여줘]: 뷰 인라인 렌더링 → "저장하시겠어요?"
```

### ChatInput 상세
```
형태: radius="xl", 멀티라인 (최대 6줄)
좌측: 📎 첨부 버튼 (ActionIcon, subtle)
      → 드롭다운: [이미지/캡처] [텍스트 붙여넣기] [파일]
우측: 전송 버튼 (텍스트 없을 때 비활성)
      스트리밍 중: ■ 중단 버튼으로 교체

키보드:
  Enter: 전송
  Shift+Enter: 줄바꿈
```

### SaveNudge 상세
```
타이밍 A (5턴 도달):
  위치: 입력창 위 sticky 배너
  "대화가 쌓이고 있어요 🌌 저장하지 않으면 사라져요"
  [Google로 저장] → OAuth → 데이터 보존 후 대화 이어짐

타이밍 B (뷰 생성 직후):
  인라인 토스트
  "저장하시겠어요?"

타이밍 C (탭 닫기 직전):
  풀스크린 모달
  "N개 노드가 생성됐어요. 저장 안 하면 사라져요."
```

### 빈 상태 (첫 방문)
```
중앙에 OU 인사말:
  "안녕하세요! 저는 OU예요.
   뭐든 말씀해보세요."

추천 입력 칩 (3개):
  [오늘 할 일 정리해줘]
  [아이디어 기록해줘]
  [다음주 계획 세워줘]
```

## 화면 4: 내 우주 `/my`

### 목적
쌓인 DataNode를 그래프와 뷰로 확인하는 홈.

### 레이아웃
```
┌── Sidebar ──┬────────────────────────────────┐
│             │  "내 우주"  [채팅으로 추가 +]   │
│             │                                 │
│             │  ── 그래프뷰 ──────────────────  │
│             │  (PixiJS, 400px 높이)           │
│             │  노드=별, 엣지=중력선           │
│             │                                 │
│             │  ── 저장된 뷰 ─────────────────  │
│             │  [카드] [카드] [카드]            │
│             │                                 │
│             │  ── 도메인별 DataView ──────────  │
│             │  schedule (N개) → CalendarView  │
│             │  task (N개) → KanbanView        │
│             │  ...                            │
└─────────────┴────────────────────────────────┘
```

### 그래프뷰 인터랙션
```
노드 클릭: 상세 사이드패널 열림 (우측, 280px)
  - domain badge
  - raw 텍스트
  - source_file_url 있으면 "원본 보기" 버튼
패닝: 마우스 드래그
줌: 마우스 휠 (0.2x ~ 3x)
노드 없음: "아직 우주가 비어있어요. 대화를 시작해보세요."
```

### 저장된 뷰 카드
```
카드 형태: 아이콘 + 이름 + 뷰 타입 badge
내 뷰: 흰 아이콘
구독 뷰: 아이콘 + @ 배지 (Phase 2)
클릭: 해당 뷰 렌더링 페이지
```

### 필터 원칙
```
DataNode 3개 미만 domain은 뷰 미표시
빈 뷰 사용자에게 절대 표시 금지
```

## 화면 6: 관리자 패널 `/admin`

### 접근 조건
```
@ouuniverse.com 이메일만 접근 가능
비관리자 접근 → /my 리다이렉트
세션 타임아웃: 30분
```

### 레이아웃
```
┌── Sidebar ──┬────────────────────────────────┐
│  Crown 아이콘│  "관리자 패널"                  │
│  (admin만)  │                                 │
│             │  통계 카드 4개                  │
│             │  [전체 회원] [DataNode수]        │
│             │  [UNRESOLVED] [오늘 API 비용]   │
│             │                                 │
│             │  Tabs:                          │
│             │  [DataNode 관리] [신뢰도 큐]    │
│             │  [비용 모니터링]                │
└─────────────┴────────────────────────────────┘
```

### DataNode 관리 탭
```
검색바 + 도메인 필터 + 신뢰도 필터
Table: 내용 | 도메인 | 신뢰도 | 생성일 | 편집
페이지네이션: 20개씩
```

### 신뢰도 큐 탭
```
verification_requests 목록
각 항목: DataNode 내용 + 투표 현황 + [승인] [거부]
항목 없음: "검토 대기 항목이 없어요 ✓"
```

### 비용 모니터링 탭
```
operation별 합계 배지
최근 50건 테이블: 작업 | 모델 | 토큰 | 비용 | 시각
비용 > $0.01 → 빨간색 강조
```


## UX 플로우 — 데이터 편집 경고

```
DataNode 상세 패널에서:
  [편집] 클릭

  일반 필드 수정 → 경고 없이 바로 편집
  태그/속성 수정 → "뷰 필터에 영향을 줄 수 있어요" [취소][변경]
  트리플/관계 수정 → "⚠️ 학습된 지식 구조가 영향받아요.
                      연결 노드 N개 영향" [취소][그래도 변경]

  [...] 메뉴:
    [오류 신고]     → 객관식 → 경고 없이 수정
    [잘못 입력했어요] → 경고 없이 수정
    [직접 수정]     → 경고 모달
```
