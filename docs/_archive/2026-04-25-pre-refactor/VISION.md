# OU — VISION.md

## 한 줄 정의

**LLM 대화를 데이터화하고, 어떤 형태의 뷰로든 꺼내 쓰는 개인 데이터 우주 플랫폼.**

메인 카피: **Just talk.**

---

## 핵심 가치

1. **대화가 곧 입력** — 키보드, 마우스, 폼이 아니라 말로 데이터를 만든다
2. **구조화는 AI가** — 유저는 자유롭게 말하고, LLM이 도메인 분류 + 구조화
3. **뷰는 무한** — 같은 데이터를 캘린더, 그래프, 사전, 마인드맵, 문서 등 어떤 형태로든 본다
4. **내 데이터 우주** — 개인이 평생 쌓는 데이터가 지식 그래프로 연결된다

---

## OU = OS

OU는 앱이 아니라 **운영체제**다.

| OS 개념 | OU 대응 | 설명 |
|---------|---------|------|
| 데스크탑 | /my 홈 | 위젯 그리드 |
| Spotlight | Orb | 대화 입력 인터페이스 |
| 파인더 | 그래프뷰 | 데이터 탐색 |
| 설정 | /settings | 시스템 설정 |
| 앱 | 뷰 (View) | DataNode 렌더링 |
| 앱스토어 + 런치패드 | Orbit | 뷰 마켓 + 뷰 모음 |
| 파일 | DataNode | 구조화된 데이터 |
| 파일시스템 | 지식 그래프 | 노드 간 관계 |

---

## API/MCP가 필요 없는 구조

기존 소프트웨어가 API와 MCP에 의존하는 이유는 **데이터가 흩어져 있어서**다.
OU는 모든 데이터가 `data_nodes` 하나에 있으므로:

- 서비스 간 API 연동 불필요 (같은 DB)
- AI ↔ 서비스 연결(MCP) 불필요 (같은 DB에서 읽고 씀)
- 데이터 마이그레이션 불필요 (뷰만 바꾸면 됨)
- 호환성 개념 자체가 불필요 (변환이 없으니 깨질 수 없음)

**OU = 데이터 통합이 아키텍처 수준에서 해결된 상태.**

---

## 뷰 = 앱. 근데 더 가볍다.

| 기존 앱 개발 | OU 뷰 개발 |
|------------|-----------|
| DB 설계 | ✗ (data_nodes) |
| 백엔드 API | ✗ (OU 파이프라인) |
| 인증/결제 | ✗ (OU Auth + 마켓) |
| 인프라/배포 | ✗ (OU가 운영) |
| 프론트엔드 전체 | 뷰 컴포넌트 1개만 |

뷰 개발자는 `ViewProps` 받아서 `nodes`를 렌더링하면 끝.
라우팅, 인증, 상태관리, DB — 전부 OU가 해준다.

앱스토어 심사/수수료 없이 Orbit에 등록 → 즉시 배포.
PWA 바로가기로 홈화면 아이콘 → 앱 설치와 동일한 경험.

---

## 그로스 전략: 뷰별 독립 앱 배포

```
Step 1: 단일 뷰로 유저 획득
  "무료 캘린더 앱" → 캘린더 뷰 아이콘 설치
  (OU인 줄 모름. 그냥 캘린더 앱인 줄 앎)

Step 2: Orb가 내장돼 있으니까
  대화할 때마다 모든 도메인 데이터가 자동 축적
  캘린더 쓰면서 지출, 감정, 할일도 기록됨 (유저 모름)

Step 3: 인라인 뷰로 자연스러운 발견
  "커피 4500원" → 💰 지출 카드 인라인 → "지출 차트 뷰 있어요"
  "오늘 좀 지침" → 😔 감정 카드 인라인 → "감정 일기 뷰 있어요"
  매번 새 도메인 데이터가 생길 때마다 Orb가 해당 뷰를 인라인으로 추천

Step 4: Orbit 노티
  "📬 투두 뷰: 할 일 3개가 이미 정리돼 있어요"
  "데이터 입력 안 했는데 이미 있네?" ← 킬러 모먼트

Step 5: 뷰 생태계 확장
  홈화면에 OU 뷰 아이콘 5개 → 전부 같은 데이터 → 전부 OU
```

**핵심: "데이터 추가 입력 안 해도 바로 쓸 수 있다."**
다른 앱은 깔면 빈 화면. OU 뷰는 이미 채워져 있다.

## 킬러 데모

### 데모 1: Just talk.
1. "내일 오후 3시에 병원 예약" → 일정 자동 생성
2. "아까 병원 시간 4시로 변경" → 기존 노드 업데이트
3. 캘린더 뷰에서 확인

### 데모 2: 한자 인라인 검색
1. 千字文 텍스트 붙여넣기 → 한자 카드 자동 표시
2. 연속 입력 → 검색 모드 전환 (LLM 스킵)
3. 카드 클릭 → 상세 정보 (읽기, 부수, 자원)

### 데모 3: 유튜브 영상 학습
1. Orb에 유튜브 URL 붙여넣기 → 영상 인라인 임베드
2. 영상 보면서 메모 → 자동 DataNode 생성
3. 커리큘럼 뷰에서 진도 관리

### 데모 4: 뷰 확장
1. 캘린더 뷰만 쓰던 유저
2. "커피 4500원" → 지출 카드 인라인 → "지출 차트 뷰 있어요"
3. 열어보니 이번 달 지출이 이미 정리돼 있음
4. "홈에 추가" → 앱 아이콘 생성

---

## OU가 아닌 것

- OU는 챗봇이 아니다 → 대화는 수단이고, 목적은 데이터화
- OU는 노트앱이 아니다 → 구조화된 데이터이지 텍스트 저장이 아니다
- OU는 특정 도메인 도구가 아니다 → 한의학, 한자, 일정, 감정 모두 같은 구조
- OU는 앱이 아니다 → 뷰가 앱이고, OU는 OS다

## 청사진

```
Phase 1: 개인 데이터 우주 (현재)
  - Orb: 대화 → 구조화 → 뷰 렌더링
  - 15개 기본 내장 뷰
  - 관리자 도메인 데이터 (한자 98K, 본초 504, 방제 200, 상한론 60)
  - /view/[viewId] PWA 바로가기

Phase 2: 뷰별 독립 앱 배포
  - 각 뷰를 독립 앱처럼 마케팅
  - Orb 내장 → 데이터 자동 축적
  - Orbit 노티 → 뷰 생태계 확장
  - 유저는 나중에야 OU를 알게 됨

Phase 3: Orbit 마켓
  - 뷰 개발자 생태계
  - 뷰 판매/구독
  - 유니(UNI) 화폐

Phase 4: 플랫폼
  - .ou 파일 포맷
  - 그룹/팀 공유
  - B2B (교육기관, 기업)
```
