### [2026-04-20] 다중 뷰 추천 + "더 많은 뷰" UX
상태: 🔄 검토중

하나의 입력에서 여러 관점의 뷰를 추천하는 기능.

예: "점심 김치찌개 12000원" 입력 시:
- 가계부 차트 (지출 12000원)
- 식단 테이블 (오늘 점심: 김치찌개)
- 메뉴 빈도 차트 (김치찌개 +1회)

구현 방향:
1. extractAll 결과에 `recommended_views: string[]` 포함 (LLM이 추천)
2. SSE done 이벤트에 recommended_views 전달
3. ChatPanel InlineView: 비중 높은 1~3개 뷰 즉시 표시
4. "더 많은 뷰를 보시겠어요?" 버튼 → 나머지 뷰 펼침
5. OrbFullscreen 좌측 패널도 동일 패턴

뷰 우선순위 기준:
- primary domain 뷰 (항상 첫 번째)
- 트리플에서 파생 가능한 뷰 (식단, 통계 등)
- 엔티티 관련 뷰 (인물 카드, 장소 등)

InlineView 리팩터 필요:
- 현재: domain별 하드코딩 switch문 → VIEW_REGISTRY inline 모드 활용으로 전환
- chatStore.nodeCreated에 recommended_views 필드 추가

선행 작업: 추출 파이프라인 아키텍처 재설계 (extraction registry)

### [2026-04-20] 스텔스 STT 모드
상태: 💡 아이디어

상시 켜두되, 쓰레기 데이터 수집과 배터리 낭비를 막기 위한 조건부 자동 활성화 STT.
회원이 손으로 켜지 않아도 대화가 자동 수집된다.

활성화 조건 (설계 필요):
- 내 목소리가 인지될 정도의 볼륨 이상
- 두 명 이상 목소리 감지 (대화 상황 판단)
- 특정 키워드 감지 (웨이크워드 방식)
- 이동 중 아님 (가속도 센서 활용)

데이터 수집 후:
- 화자 분리 (내 목소리 vs 상대방)
- 주요 내용 추출 → DataNode 생성
- 대화 상대 태깅

"오늘 있었던 일" 탭:
- 자기 전 하루 대화 데이터를 타임라인으로 요약
- Accuracy 역할: 잘못된 데이터 수정, 불필요한 데이터 삭제 (회원 UI 기준)
- OU 내부에는 원본 전부 보존 (데이터 불삭제 원칙 유지)
- 회원 편집 → 데이터 품질 자연스럽게 향상 (인간 피드백 루프)

Accuracy 감정 필터링:
- 수집된 대화의 감정 톤 분석 (부정/갈등/슬픔 등 감지)
- 민감도 높은 내용은 "오늘 있었던 일" 탭에 자동 제외 또는 흐리게 처리
- 회원이 직접 열어보는 방식 ("숨겨진 항목 보기") 으로만 접근
- 목적: 나쁜 기억을 리마인드시켜 역효과 유발 방지
- OU 내부에는 전부 저장 (원본 보존 원칙) — 노출 여부만 필터링

프라이버시 원칙:
- 수집 중 시각적 인디케이터 필수 (스텔스이되 회원 자신은 인지)
- 로컬 처리 우선, 서버 전송은 회원 명시 동의 후
- 수집 조건 커스터마이징 가능 (민감도 조절)

### [2026-04-20] 스와이프 Accuracy + 유니 보상
상태: 💡 아이디어

Accuracy를 무거운 작업이 아니라 틴더식 스와이프 UX로 전환.
데이터 카드가 뜨면 → 맞으면 오른쪽, 틀리면 왼쪽. 3초면 1개 정제 완료.

유니(UNI) 보상:
- 스와이프 1회 = 유니 적립
- 연속 정제 시 보너스 (스트릭)
- 유니로 프리미엄 뷰 구매 / Orbit 마켓 이용

선순환 구조:
스와이프(재미) → 유니 적립(보상) → 뷰 구매(소비) → 더 좋은 데이터 위에서 뷰 작동(가치) → 더 스와이프하고 싶어짐

핵심: Accuracy가 "해야 하는 일"이 아니라 "심심할 때 하는 놀이"가 됨

### [2026-04-20] 프로필 노드 — 대화형 소셜 그래프
상태: 💡 아이디어

페이스북의 소셜 그래프를 대화만으로 만드는 구조.

프로필 노드 = 사람에 대한 모든 정보를 담는 특별한 DataNode:
- 내 프로필 노드: 회원 자신의 정보
- 친구 프로필 노드: 대화 속에서 자연스럽게 축적

저장 필드 (대화에서 자동 추출):
- 이름, 번호, 주소, 메일, 생일, MBTI
- 좋아하는 것, 싫어하는 것
- 관계 (친구/가족/직장 등)
- 기타 대화에서 언급된 모든 정보

핵심 차별점:
- 페이스북: 친구가 직접 프로필을 채워야 함 → 안 하면 빈 껍데기
- OU: 내가 대화하면서 자연스럽게 쌓임 → 관심 많을수록 풍성해짐
- 친할수록 프로필이 풍성 = "관심의 시각화"

유니버스 연결:
- 친구가 OU 가입 시 → 내가 만든 친구 프로필 노드 ↔ 친구의 실제 유니버스 링크
- 양방향 데이터 연결 (권한 기반)
- 친구가 공개한 DataNode를 내 그래프에서 탐색 가능

파생 기능:
- 관계 메모리: "OO이 좋아하는 거 뭐였지?" → 프로필 카드 한 장으로 해결
- 선물 추천, 생일 알림
- 소셜 그래프 시각화 (그래프뷰에서 인물 관계망)

프로필 공유 프라이버시 원칙 (불변):
- 기본값 = 완전 비공유. 자동 공유 절대 금지
- 데이터 분류:
  - 팩트 (이름, 번호, 생일, 혈액형 등): 공유 후보. 받는 사람(B)이 항목별 선택
  - 감정/평가/메모 ("답답해", "좋은 사람"): 공유 대상에서 영구 제외. A의 유니버스 안에서만 존재
- "이 정도는 괜찮겠지" 판단 금지 — 사람마다 민감도 기준 다름
- 공유 주도권 = 받는 사람(B): B가 가입 시 항목별 체크로 수락
- 공유 트리거 = A가 B에게 프로필 카드(링크)를 직접 공유하는 행위
  - A의 공유 행위 자체가 동의 (시스템이 자동 판단하지 않음)
  - A가 공유할 항목을 직접 선택 (보여주고 싶은 팩트만)
  - B가 링크 클릭 → 가입 → A가 공유한 정보만 자동 등록
  - 바이럴 효과: 프로필 카드 공유 = 자연스러운 초대 메커니즘
- A가 공유하지 않은 데이터(감정/메모/평가)는 A의 유니버스에만 영구 존재

콜드 스타트 해결 — 컨텍스트 기반 온보딩:
- B 가입 시 이미 프로필 데이터가 존재 → 빈 우주가 아닌 상태에서 시작
- 튜토리얼이 B의 데이터를 활용한 개인화 대화로 진행
  예: "희민님, 생일이 5월 3일이시네요! 다가오는 생일 준비해볼까요?"
- 일반적인 "여기 누르세요" 가 아니라 "OU가 나를 아는" 첫인상
- 효과: 콜드 스타트 제거 + 즉시 가치 체감 + 초기 이탈 방지

초대 유니 보상 (그로스 루프):
- 프로필 카드 공유 → 친구 가입 시 A에게 유니 적립
- "초대하세요" 가 아니라 "카드 공유" 라는 자연스러운 행위가 초대
- 완전한 루프: 대화 → 프로필 생성 → 카드 공유 → 가입 → 유니 보상 → 온보딩 → 대화 → 반복
- 유니 = 투자비용 대체: 기프티콘(현금 유출) 대신 유니(생태계 순환)로 그로스 비용 조달
  - 일반 스타트업: 현금 → 기프티콘 → 가입 → 이탈 → 돈 날림
  - OU: 유니 → 보상 → 가입 → 유니 쓰려면 서비스 사용 → 락인
  - 유니가 뿌려질수록 생태계 강화 (현금 유출 제로)

기존 인프라: domain='relation' 이미 존재 → 프로필 노드로 확장

### [2026-04-20] 알림톡 → 자동 배송추적 뷰
상태: 💡 아이디어

카카오 알림톡(택배 발송 알림)이 Orb로 들어오면 → 주문번호, 송장번호, 택배사 자동 추출 → 배송추적 뷰 자동 생성.

현재 세상: 알림톡 → 눈으로 읽기 → 송장번호 복사 → 택배사 사이트 이동 → 붙여넣기 → 조회
OU 세상: "굽네몰 배송 어디야?" → 끝

배송 차량 GPS 연동:
- 택배사 API + 차량 GPS(지오코드) 데이터 연동
- 카카오맵 타일 위에 OU 커스텀 오버레이로 차량 실시간 위치 표시
- 출발지→현재위치→도착지 폴리라인 경로
- OU 디자인 시스템 적용 (회원은 카카오맵인 줄 모름)
- 선행: 지도뷰 (이미 계획됨)

파생:
- 전체 주문 타임라인 뷰 (주문→발송→배송중→도착)
- "오늘 올 택배 있어?" → 한 눈에 확인

### [2026-04-20] 브레인스토밍 — 향후 탑재 기능 목록
상태: 💡 아이디어

수집의 진화 (Orb 확장):
- 스텔스 STT (별도 항목)
- 스크린 리딩: 브라우저/앱에서 보고 있는 콘텐츠 백그라운드 수집
- 사진 한 장이면 끝: 메뉴판/명함/영수증/칠판 → Gemini Vision → DataNode
- 위치 기반 자동 태깅: GPS + 대화 연결 → "그 카페에서 뭐 이야기했지?"
- 웨어러블 연동: 워치/이어버드 상시 수집, 바이오 데이터 연결

기억의 진화 (Recall):
- 타임 트래블: "3개월 전 나 뭐하고 있었지?" → 시점별 타임라인 재생
- 망각 곡선 알림: 에빙하우스 기반, 잊을 타이밍에 리마인드 (감정 필터링 적용)
- "내가 뭐라 했더라?": 내가 한 약속/발언 추적
- 모순 감지: 과거 발언과 현재 발언 차이 감지 → 생각 변화 인지

이해의 진화 (OU가 나를 안다):
- 패턴 디텍션: 회원 자신도 몰랐던 행동/감정 패턴 발견
- 의사결정 서포트: 고민 시 관련 과거 발언 모아서 보여줌
- 관심사 그래프: 관심사 변화 시각화

관계의 진화:
- 사람 노드: 대화 속 인물 자동 노드화
- 관계 메모리: 타인의 취향/정보 자동 추출
- 선물 추천: 관계 메모리 기반

하루의 진화:
- 오늘 있었던 일 (별도 항목)
- 위클리/먼슬리 다이제스트
- 연말 회고 자동 생성 (스포티파이 Wrapped 형태, 공유 가능)

Accuracy의 진화:
- 감정 필터링 (별도 항목)
- 스와이프 Accuracy + 유니 보상 (별도 항목)
- 학습 모드: 회원 정정 패턴에서 OU가 학습, 같은 실수 반복 방지
