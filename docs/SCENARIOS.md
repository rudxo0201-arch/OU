# SCENARIOS.md — 페르소나별 UX 시나리오

> 각 페르소나가 OU를 어떻게 경험하는지 구체적 대화 예시와 함께 정리.
> 마케팅, UX 기획, 온보딩 설계 시 참조.

## 1. 첫 방문자 — "이게 뭐지?"

**상황**: 링크를 타고 ouuniverse.com에 처음 접속

```
→ 랜딩페이지. 채팅창이 바로 보임
→ "다음주 일요일 희민이 결혼식" 입력
→ 그래프에 캘린더 노드가 흐릿하게 생성됨
→ "💫 일정이 생성됐어요"
→ "나 결혼식 언제였지?" 질문 → 캘린더뷰가 렌더링됨
→ 5턴쯤 "대화가 쌓이고 있어요. 사라지기 전에 저장하세요"
→ [저장하기] → Google 원클릭 가입
→ "아까 하던 얘기 계속해요 😊"
```

**핵심 감정**: "가입도 안 했는데 벌써 데이터가 생겼네? 잃고 싶지 않다"

---

## 2. 일상 기록자 (직장인) — "앱 5개가 채팅 하나로"

**상황**: 매일 채팅창에 떠오르는 것을 입력하는 직장인

```
아침: "오늘 팀 미팅 3시, 발표자료 준비해야 함"
     → schedule + task DataNode 자동 생성

점심: "점심 12,000원 김치찌개"
     → finance DataNode 생성

저녁: "민준이가 이직한다고 했어. 네이버로 간대"
     → relation DataNode (민준-이직-네이버) 생성

주말: "이번 달 뭐에 돈 썼지?"
     → 차트뷰 자동 렌더링 (식비/교통/기타 파이차트)
```

**핵심 감정**: "앱 5개 쓰던 걸 채팅 하나로 다 하고 있다"

---

## 3. 수험생 — "공부 데이터가 쌓인다"

**상황**: 시험을 준비하는 학생

```
"이 문제 풀이 설명해줘" + 사진 촬영
→ OU가 풀이 설명 + knowledge DataNode 생성
→ 틀린 문제 자동으로 오답 노드 축적

2주 뒤: "최근에 틀린 문제 보여줘"
→ 오답노트 뷰 자동 렌더링

1달 뒤: "오답 많은 단원이 뭐야?"
→ 차트뷰: 단원별 오답률 시각화

시험 전: "오답노트 PDF로 뽑아줘"
→ 출력 뷰로 PDF 생성 → 인쇄
```

**핵심 감정**: "오답노트를 따로 안 만들어도 되네"

---

## 4. 기획자/창작자 — "아이디어가 사라지지 않는다"

**상황**: 신규 서비스를 기획하는 PM

```
"사용자가 사진 올리면 AI가 분류해주는 서비스 어때?"
→ idea DataNode 생성

"경쟁사 분석해봤는데 A사는 이런 식이고 B사는..."
→ knowledge DataNode + 관계 트리플 자동 생성

"오늘 기획 대화에서 나온 아이디어 보여줘"
→ 마인드맵뷰 렌더링 (아이디어 간 연결 시각화)

"이거 기획서로 뽑아줘"
→ Docs 출력 뷰 → 기획서 자동 생성
```

**핵심 감정**: "대화한 게 그대로 문서가 됐다. 정리 시간이 사라졌다"

---

## 5. 도메인 학습자 — "교과서가 살아있는 그래프"

> 예시: 한의대생이 관리자 구축 본초/방제 DB를 구독.
> 어떤 도메인이든 같은 패턴으로 작동한다.

```
"마황의 효능 알려줘"
→ 도메인 특화 뷰 렌더링 (기원, 분류, 속성 구조화)

"마황이 포함된 조합 전부 보여줘"
→ 관련 DataNode 필터링

"감기 관련 항목들 관계도 보여줘"
→ 그래프뷰: 관련 노드가 별처럼 연결

시험 전: "이 범위 퀴즈 내줘"
→ 퀴즈뷰 자동 생성 (DataNode 기반)

틀린 문제 → 오답 DataNode → "오답만 모아줘" → PDF
```

**핵심 감정**: "교과서 + 오답노트 + 퀴즈앱이 하나다"

---

## 6. 크리에이터/인플루언서 — "콘텐츠 한 번이 무한 재활용"

**상황**: 전문가가 자신의 DataNode를 유료 공개

```
강의 녹음 파일 업로드
→ STT → 마크다운 교재 자동 생성 → knowledge DataNode

"3강까지 내용 챗봇으로 열어줘"
→ 챗봇뷰 생성 (구독자가 강사에게 직접 질문하는 경험)
→ 구독자만 접근 가능

구독자 A: "3강에서 말한 내용 다시 설명해줘"
→ 챗봇뷰가 강사 DataNode 기반으로 답변

구독자 B: "강의 내용 전자책으로 볼 수 있나요?"
→ 전자책뷰(.epub) 렌더링 → 구독자 열람
```

**핵심 감정**: "콘텐츠 한 번 만들면 뷰만 바꿔서 무한 재활용"

---

## 7. 가계부 관리자 — "말하면 가계부"

**상황**: 매일 지출을 채팅으로 기록

```
"커피 4,500원" "택시 12,000원" "월급 350만원 들어옴"
→ finance DataNode 자동 분류 (수입/지출/카테고리)

"이번 달 식비 얼마 썼어?"
→ 차트뷰 자동 렌더링

"지난 3개월 지출 추이 보여줘"
→ 선 그래프 뷰

"이번 달 예산 넘었어?"
→ 예산 대비 현황 비교
```

**핵심 감정**: "가계부 앱 깔 필요가 없다"

---

## 8. 인간관계 관리자 — "사람이 기억된다"

**상황**: 사람 관련 대화가 자연스럽게 축적

```
"오늘 수진이랑 밥 먹었는데 이직 고민 중이래"
→ relation DataNode (수진-이직고민) + schedule

3개월 뒤: "수진이 근황 뭐였지?"
→ 수진 관련 DataNode 전부 타임라인뷰로 렌더링
→ "4월: 이직 고민, 6월: 새 회사 입사"

"내 인맥 관계도 보여줘"
→ 관계 그래프뷰: 사람 노드가 별처럼 연결
```

**핵심 감정**: "사람 이름 하나로 모든 맥락이 나온다"

---

## 9. 독서가 — "읽은 책이 지식이 된다"

**상황**: 책 읽고 메모를 채팅으로 남김

```
"'사피엔스' 3장 읽었는데, 인지혁명 핵심은..."
→ knowledge DataNode (사피엔스-인지혁명)

"예전에 읽은 '총균쇠'에서도 비슷한 얘기 나왔는데"
→ 트리플 자동 생성: (인지혁명, related_to, 지리적결정론)

"올해 읽은 책 보여줘"
→ 책장뷰(갤러리) 렌더링

"읽은 책들 사이에 공통점 있어?"
→ 그래프뷰: 책 간 개념 연결 시각화
```

**핵심 감정**: "메모가 그냥 쌓이는 게 아니라 서로 연결된다"

---

## 10. 프로필 관리자 — "같은 나, 다른 뷰"

**상황**: 같은 데이터를 맥락에 따라 다른 프로필로

```
취업 면접용: "내 경력 프로필뷰 만들어줘"
→ 경력/프로젝트 DataNode → 경력 프로필 스타일 렌더링

동아리 소개용: "내 관심사 그래프뷰로 공유할래"
→ 취미/지식 DataNode → 그래프뷰 공유 링크 생성

블로그용: "내 독서 기록 타임라인으로 공개"
→ 독서 DataNode → 타임라인뷰 웹 공개
```

**핵심 감정**: "같은 나인데 보여주고 싶은 면만 골라서 보여줄 수 있다"

---

## 11. 대학생 — "캠퍼스 생활 전체가 채팅 하나"

**상황**: 시간표, 과제, 가계부, 진로를 따로따로 관리하던 대학생

```
📅 시간표/일정
"내일 경영학 10시, 통계 2시" → 캘린더뷰
"목요일 MT 강원도" → schedule DataNode

📝 수업 메모
"교수님이 중간고사 범위 1~5장이래" → knowledge DataNode
"오늘 마케팅 수업 핵심: STP 전략이란..." → 자동 구조화
→ "마케팅 수업 내용 정리해줘" → 시험 전 자동 요약

💰 생활비
"편의점 3,200원" "교재 28,000원" "알바비 95만원"
→ "이번 달 남은 돈 얼마야?" → 차트뷰

📚 시험 준비
틀린 문제 사진 촬영 → 오답 DataNode
→ "통계 오답만 모아줘" → 퀴즈뷰로 복습
→ "PDF로 뽑아줘" → 시험 직전 오답노트

🤝 인간관계
"오늘 수진이 생일" "민호가 동아리 탈퇴한대"
→ 3개월 뒤 "수진이 생일 언제였지?" → 즉시 답변

💡 진로
"인턴 지원해야 하는데 자소서 소재가..."
→ 그동안 쌓인 경험/프로젝트 DataNode → 포트폴리오뷰
```

**핵심 감정**: "앱 10개 쓰던 걸 채팅 하나로. 졸업 후에도 내 우주는 계속된다"

**킬러 포인트**: 4년간 쌓인 대학생활 데이터가 졸업 후에도 내 우주로 남는다. 노션은 졸업하면 버린다. OU는 인생 데이터가 계속 이어진다.

## 12. 육아맘 — "정신없는 하루를 말로만 기록"

**상황**: 손이 자유롭지 않은 육아 중. 앱을 열고 버튼 누를 여유가 없다.

```
🍼 수유/이유식
"7시에 분유 180ml" "12시 이유식 소고기미음"
→ 수유 기록 DataNode 자동 축적
→ "오늘 분유 총 얼마나 먹었어?" → 즉시 답변
→ "이번 주 수유 패턴 보여줘" → 차트뷰

😴 수면
"낮잠 1시~3시" "밤에 3번 깼어"
→ "수면 패턴 보여줘" → 히트맵뷰
→ 소아과 갈 때 "최근 2주 수면 기록 보여줘" → 의사에게 바로 보여줌

🏥 성장/병원
"6개월 검진 7.8kg" "접종 BCG 완료"
→ 성장 기록 DataNode
→ "접종 뭐 남았어?" → 리스트뷰
→ "성장 곡선 보여줘" → 차트뷰

📸 성장 일기
"오늘 처음 뒤집었어!" "첫 이유식 반응 웃김"
→ emotion + milestone DataNode
→ "첫돌까지 성장 타임라인 보여줘" → 타임라인뷰
→ 전자책뷰로 → 육아 포토북 자동 생성

🛒 육아 정보
"기저귀 뭐가 좋대? 하기스 vs 팸퍼스"
→ knowledge DataNode
→ 나중에 둘째 때 "첫째 때 기저귀 뭐 썼지?" → 즉시

😮‍💨 감정 기록
"오늘 진짜 힘들었어. 아기가 3시간 울었어"
→ emotion DataNode
→ "최근 한 달 내 감정 흐름 보여줘" → 저널뷰
→ 힘든 날과 좋은 날의 패턴 발견
```

**핵심 감정**: "손이 자유롭지 않아도 말만 하면 다 기록된다"

**킬러 포인트**:
- 손이 자유롭지 않다 → 음성으로 "분유 180ml"만 말하면 끝
- 기억력이 떨어진다 (수면 부족) → OU가 다 기억해줌
- 소아과 갈 때 → "최근 기록 보여줘" 한마디로 의사에게 보여줌
- 아이가 크면 → 몇 년간 쌓인 데이터가 성장 기록 자체가 됨

## 풀 퍼널: 비로그인 → 습관 → 유료 전환

```
[비로그인] 채팅 체험 → 노드 생성 확인 → 사라지기 전 가입
[Free]     매일 채팅 → 데이터 축적 → 뷰 추천 수락 → 우주가 커짐
[전환점]   "이 뷰를 더 예쁘게 만들고 싶다" or "PDF로 뽑고 싶다"
[Pro]      AI 뷰 생성 / 고급 출력 / 챗봇뷰 구독 판매
```

핵심: 무료로 데이터가 충분히 쌓인 뒤, "이걸 더 잘 활용하고 싶다"는 자발적 욕구에서 전환

## 공통 관통 경험

| 원칙 | 사용자가 느끼는 것 |
|------|-------------------|
| **Just talk** | 앱을 배울 필요 없다. 말하면 된다 |
| **세션 없음** | 어제 대화든 3개월 전이든 이어진다 |
| **자동 구조화** | 정리를 안 해도 알아서 분류됐다 |
| **뷰 전환** | 같은 데이터를 표/그래프/PDF/퀴즈로 본다 |
| **우주가 커짐** | 쓸수록 내 데이터 우주가 풍부해진다 → 떠날 수 없다 |
