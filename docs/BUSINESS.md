## 마케팅 전략

### OU 마케팅의 핵심 원칙

```
말로 설명하는 마케팅 ❌
직접 경험시키는 마케팅 ✅

"LLM 대화가 데이터가 된다"는
설명으로는 이해 안 됨. 한 번 경험해야 함.
킬러 데모가 전환을 결정한다.
```

### 타겟 페르소나

```
A) LLM 헤비유저
   Claude/GPT 매일 쓰는데 대화가 다 날아감
   "어디서 봤더라" 매번 반복
   → 즉시 공감

B) 정보 과부하 겪는 사람
   노션 만들어놓고 안 씀
   메모앱 카오스
   아이디어가 어딘가 있는데 못 찾음
   → 즉시 공감

C) 지식 크리에이터
   강의, 책, 뉴스레터 만드는 사람
   콘텐츠 재활용이 안 됨
   DataView가 솔루션

D) 기획자/창업자/연구자
   브레인스토밍이 기록이 안 됨
   대화하다 나온 아이디어가 사라짐
   → 이 대화 자체가 페인포인트 (OU의 킬러 유즈케이스)

E) 학생/직장인
   일정, 과제, 할 일 앱이 따로따로
   켜고 싶지 않음
   → Just talk. 해결
```

## 교육 시장 전략

### 왜 교육인가

```
기존 교육 콘텐츠의 한계:
  강의자료 → 일방적 전달 → 저장 → 잃어버림
  문제지: 풀면 끝. 데이터 안 남음.
  오답노트: 직접 만들어야 함.

OU:
  모든 교육 콘텐츠 = DataNode
  모든 교육 결과물 = 데이터뷰
  데이터가 살아있음 → 업데이트, 연결, 진화
```

### 강의 시장

```
강사:
  강의 DataNode 구축 (직접 입력 or 파일 업로드 → 자동 변환)
  강의뷰 설정 → 수강생에게 구독 링크 배포

수강생:
  구독 → 강의뷰 수신 (항상 최신)
  자신의 노트 DataNode 추가
  → 강의 + 내 노트 = 하나의 우주
  같은 강의를 각자 원하는 뷰로:
    요약 카드뷰 / 마인드맵뷰 / 퀴즈뷰 (자동 생성)

언어 중립:
  강사 언어로 작성 → 수강생 언어로 렌더링
  → 글로벌 강의 플랫폼 가능
```

### 그래프뷰가 강의 콘텐츠 생성기가 된다

```
강사가 DataNode를 쌓으면
그래프뷰가 강사도 몰랐던 연결을 보여준다.

예시 (어떤 도메인이든):
  마케터:   광고/채널/전환율 DataNode 쌓음
            → 채널 간 연결 발견 → 새 강의 주제

  개발자:   기술/패턴/언어 DataNode 쌓음
            → 예상 못한 연결 발견 → 새 콘텐츠

  요리 강사: 재료/레시피/영양 DataNode 쌓음
             → 재료 간 관계 → 새 레시피 강의

강의 콘텐츠 = 데이터 × 연결 × 시각화 = 무한 생성
DataNode 쌓을수록 강의 주제 늘어남 → 강사 플라이휠
```

### 문제지 시장

```
현재 문제지 시장: 단원 고르면 자동으로 PDF 생성
OU: 이 시장도 흡수 가능. 문제지도 "뷰"이기 때문.

문제 DataNode → 어떤 뷰로든:
  PDF뷰:          문제지
  퀴즈뷰:         인터랙티브 풀기
  오답 DataNode:  자동 생성
  오답 PDF뷰:     오답노트 자동 생성
  취약점 그래프뷰: "내가 틀리는 유형" 시각화
  맞춤 문제지:    취약 단원만 추출 → PDF

데이터는 하나, 뷰는 무한.
풀수록 데이터 쌓임 → 더 정교한 맞춤 문제지.
```

### 교육 생태계

```
생산자: 강사(강의뷰 구독 판매), 선생님(문제지 판매), 출판사
소비자: 학생, 학부모(리포트뷰), 학원(대시보드뷰)
OU: 모든 거래 수수료 + DataNode 축적으로 이탈 불가

확장 경로 (어떤 분야든):
  전문직 강의 → 일반 교육 → 글로벌
  언어 중립 렌더링 → 해외 진출

도메인 예시 (제한 없음):
  프로그래밍, 디자인, 마케팅, 법률, 의학,
  요리, 음악, 언어, 수학, 역사... 모든 분야
```

## 게임화: 우주 계급 시스템

### 개념

```
데이터를 쌓을수록 우주 계급이 올라간다.
데이터의 양과 품질을 평가해 칭호 부여 + 수익화 권한 해금.
게임하듯 열심히 데이터를 모으게 만드는 구조.
```

### 계급 체계

```
Stardust   → Asteroid → Planet → Solar System →
먼지          소행성      행성     태양계

Nebula     → Galaxy → Galaxy Cluster → Universe
성운          은하     은하단            우주
```

### 평가 지표

```
양적 지표:
  DataNode 수, Triple 수, 도메인 다양성, 기여 DataNode 수

질적 지표:
  정확도 검증률, 커뮤니티 참조 수 (다른 사용자가 구독/인용),
  출처 품질, 집단지성 검토 기여도
```

### 계급별 권한 해금

```
Stardust ~ Planet: 기본 사용
Solar System:      공개 DataView 제작 가능
Nebula:            콘텐츠 판매 가능 (구독 판매, 조회수 수익)
Galaxy:            강사 등록 가능 (강의뷰 구독 판매)
Galaxy Cluster:    마켓플레이스 입점 (템플릿/스킨 판매)
Universe:          OU 공식 파트너 (B2B 협업, 수수료 우대)
```

### 핵심 원칙

```
칭호는 프로필에 표시 → 신뢰 지표로 작동
계급 하락 없음 → 쌓은 것은 영구 보존 (데이터 불삭제 원칙과 동일)
계급 올리는 것 = 수익 올리는 것 → 자발적 데이터 생산 플라이휠
```

## 관리자 기본 DB 확충

### 목적

```
1) LLM 정확도 향상
   기본 DB 없어도 LLM이 채움 (confidence: 'medium')
   기본 DB 있으면 → confidence: 'high' (검증된 앵커)

2) 사용자 데이터 입력 장벽 낮춤
   "사피엔스 읽을까 해" → 저자/장르/핵심 개념 자동 매핑
   사용자가 짧게 말해도 시스템이 채워줌
```

### 구축 우선순위

```
1순위 (범용): 도서, 인물, 개념, 장소, 대중콘텐츠
2순위 (도메인): 각 분야 전문 DB (확장 가능 구조)
3순위 (자동): UNRESOLVED 엔티티 → 자동 추가 큐

도메인은 특정 분야에 종속되지 않는다.
사용자가 많이 쓰는 분야부터 자동으로 확충.
```

### 집단지성 검토 시스템

```
의심 DataNode → 관련 관심사 가진 회원에게 검토 요청
객관식 투표 (맞아요 / 틀려요 / 모르겠어요)
N명 승인 → confidence = 'high' 자동 복원
미해소 → 관리자 최종 검토 (최소화)

장점:
  해당 분야 전문가가 자연스럽게 검토
  관리자 혼자 감당 불필요
  커뮤니티 소속감 + 기여도
  미래: 검토 기여 → 인센티브 연결
```
