# OU — VISION.md

## 한 줄 정의

**LLM 대화를 데이터화하고, 어떤 형태의 뷰로든 꺼내 쓰는 개인 데이터 우주 플랫폼.**

메인 카피: **Just talk.**

---

## 핵심 가치

1. **대화가 곧 입력** — 키보드, 마우스, 폼이 아니라 말로 데이터를 만든다
2. **구조화는 AI가** — 유저는 자유롭게 말하고, LLM이 도메인 분류 + 구조화
3. **뷰는 무한** — 같은 데이터를 캘린더, 그래프, 사전, 마인드맵, 문서 등 어떤 형태로든 본다
4. **내 데이터 우주** — 개인이 평생 쌓는 데이터가 지식 그래프로 연결된다

---

## 킬러 데모

### 데모 1: Just talk.
1. "내일 오후 3시에 병원 예약" → 일정 자동 생성
2. "아까 병원 시간 4시로 변경" → 기존 노드 업데이트
3. 캘린더 뷰에서 확인

### 데모 2: 한자 인라인 검색
1. 千字文 텍스트 붙여넣기 → 한자 카드 자동 표시
2. 연속 입력 → 검색 모드 전환 (LLM 스킵)
3. 카드 클릭 → 상세 정보 (읽기, 부수, 자원)

### 데모 3: 도메인 데이터 탐색
1. 관리자가 구축한 본초(한약재) 504종 그래프
2. 부수별 98,682자 한자 그래프
3. 누구나 자기 도메인 데이터를 같은 방식으로 구축

---

## OU가 아닌 것

- OU는 챗봇이 아니다 → 대화는 수단이고, 목적은 데이터화
- OU는 노트앱이 아니다 → 구조화된 데이터이지 텍스트 저장이 아니다
- OU는 특정 도메인 도구가 아니다 → 한의학, 한자, 일정, 감정 모두 같은 구조

---

## 청사진

```
Phase 1: 개인 데이터 우주 (현재)
  - 대화 → 구조화 → 뷰 렌더링
  - 관리자 도메인 데이터 (한자, 본초, 방제)

Phase 2: 소셜 + 마켓
  - 그룹 공유, 뷰 마켓, 구독
  - 페르소나별 공개 범위

Phase 3: 플랫폼
  - 뷰 = SaaS
  - .ou 파일 포맷
  - 뷰 개발 생태계
```
